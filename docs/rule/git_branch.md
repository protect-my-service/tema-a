# 🌿 Git 브랜치 전략

## 브랜치 네이밍 규칙

```
{이름}/{의도를 표현}
```

## 예시

| 브랜치명              | 설명        |
|-------------------|-----------|
| `yoonhwan/attack` | 공격시나리오 작성 |
| `junseong/attack` | 공격시나리오 작성 |
| `sojin/attack`    | 공격시나리오 작성 |
| `sora/attack`     | 공격시나리오 작성 |

## 규칙

- `main` 브랜치에 직접 push 금지
- 작업 완료 후 PR을 통해 팀원 리뷰어 2명 이상이 승인시 머지 가능
- 머지 완료된 브랜치는 삭제
